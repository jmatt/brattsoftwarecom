<p><b>Freeware</b> (from "free" and "software") is <a href="/wiki/Computer_software">computer software</a> that is available for use at no cost or for an optional fee,<sup id="cite_ref-0" class="reference"><a href="#cite_note-0"><span>[</span>1<span>]</span></a></sup> but usually with one or more restricted usage rights.<sup id="cite_ref-linfo_1-0" class="reference"><a href="#cite_note-linfo-1"><span>[</span>2<span>]</span></a></sup><sup id="cite_ref-lawrence_2-0" class="reference"><a href="#cite_note-lawrence-2"><span>[</span>3<span>]</span></a></sup> Freeware is in contrast to <a href="/wiki/Commercial_software">commercial software</a>, which is typically sold for profit. The term does not imply that the software is <a href="/wiki/Free_and_open_source_software">free and open source software</a> (FOSS). Freeware is a loosely defined category, which includes both closed and open source <a href="/wiki/Proprietary_software">proprietary software</a>.</p>

<p>The term <i>freeware</i> was coined by <a href="/wiki/Andrew_Fluegelman">Andrew Fluegelman</a> when he wanted to sell a communications program named <a href="/wiki/PC-Talk">PC-Talk</a> that he had created but for which he did not wish to use traditional methods of distribution because of their cost.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span>[</span>4<span>]</span></a></sup> Fluegelman actually distributed PC-Talk via a process now referred to as <a href="/wiki/Shareware">shareware</a>. Current use of the term freeware does not necessarily match the original concept by Andrew Fluegelman.</p>
<h2><span class="editsection">[<a href="/w/index.php?title=Freeware&amp;action=edit&amp;section=2" title="Edit section: Criteria">edit</a>]</span> <span class="mw-headline" id="Criteria">Criteria</span></h2>
<p>Software classified as freeware is licensed at no cost and is either fully functional for an unlimited time; or has only basic functions enabled with a fully functional version available commercially or as shareware.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>5<span>]</span></a></sup> In contrast to <a href="/wiki/Free_software">free software</a>, the author usually restricts one or more rights of the user, including the rights to copy, distribute, modify and make derivative works of the software or extract the source code.<sup id="cite_ref-lawrence_2-1" class="reference"><a href="#cite_note-lawrence-2"><span>[</span>3<span>]</span></a></sup><sup id="cite_ref-linfo_1-1" class="reference"><a href="#cite_note-linfo-1"><span>[</span>2<span>]</span></a></sup><sup id="cite_ref-adobe-reader_5-0" class="reference"><a href="#cite_note-adobe-reader-5"><span>[</span>6<span>]</span></a></sup><sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span>[</span>7<span>]</span></a></sup> The <a href="/wiki/Software_license">software license</a> may impose additional restrictions on the type of use including personal use, private use, individual use, non-profit use, non-commercial use, academic use, educational use, use in charity or humanitarian organisations, non-military use, use by public authorities or various other combinations of these type of restrictions.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span>[</span>8<span>]</span></a></sup> For instance, the license may be "free for private, non-commercial use". The software license may also impose various other restrictions, such as restricted use over a network, restricted use on a server, restricted use in a combination with some types of other software or with some hardware devices, etc.<sup id="cite_ref-adobe-reader_5-1" class="reference"><a href="#cite_note-adobe-reader-5"><span>[</span>6<span>]</span></a></sup></p>
<p>The <a href="/wiki/Free_Software_Foundation">Free Software Foundation</a> (FSF) has requested that people avoid referring to true Free Software as "freeware".<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span>[</span>9<span>]</span></a></sup><sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span>[</span>10<span>]</span></a></sup> Members of FOSS community believe that "freedom to use" implies that the end user is free run, and study, and modify, and distribute the software with minimal restriction.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span>[</span>11<span>]</span></a></sup> Freeware is also distinct from <a href="/wiki/Shareware">shareware</a>; the latter obliges the user to pay after some trial period or to gain additional functionality.<sup id="cite_ref-linfo_1-2" class="reference"><a href="#cite_note-linfo-1"><span>[</span>2<span>]</span></a></sup></p>
